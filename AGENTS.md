# AGENTS.md

- This repo is a Quickshell config for `niri`; the entrypoint is `shell.qml`.
- `shell.qml` instantiates `NotificationsDaemon`, `IdleManager`, and one `Bar` plus one `Background` per screen.
- `Modules/Background.qml` reads the wallpaper from `~/.config/background` and reloads it when the file changes.
- `Modules/Bar/` holds the status bar; `Modules/Notifications/` holds the notification daemon and popup UI; `Commons/` holds shared theme singletons like `Color.qml` and `Style.qml`.
- Before changing anything Quickshell-related, check the Quickshell v0.3.0 docs first: `https://quickshell.org/docs/v0.3.0/types/`.
- Keep QML changes minimal and consistent with the existing import namespaces (`qs.Commons`, `qs.Modules.*`).
- There are no repo-local build or test scripts in the tracked files; rely on the Quickshell runtime for validation.

## QML linting

- Before considering any touched QML file correct, run `qmllint` on it.
- Before linting, take the import paths from `.qmlls.ini` and pass them to `qmllint` with `-I` because the Quickshell build path changes between restarts.
- `.qmlls.ini` is generated by the shell setup in `shell.nix`; do not edit it by hand.
- If `.qmlls.ini` is missing, Quickshell is probably not running or the shell has not been initialized yet.
- If you need `python3`, run it through `nix-shell`.
